# 클래스 디자인 & 사용자 인터페이스 설명

> 전자봉투 기반 익명 동료평가 및 민감 평가 제출 시스템 (Java 콘솔 프로토타입)
> PDF 산출물 #3 — (optional) 클래스 디자인과 사용자 인터페이스에 대한 간단한 설명

---

## 1. 클래스 디자인

### 1.1 설계 원칙

- **기능별 분리**: 키 관리 / 대칭암호 / 전자봉투 / 전자서명 / 파일 I/O 를 각각 독립 클래스로 분리 (단일 책임).
- **유틸리티 + 실행부 분리**: 5개의 기능 클래스(`*Util`, `CryptoUtil`, `MyKeyPair`)와 2개의 실행 프로그램(`EvaluationSubmit`, `EvaluationRead`)으로 구성.
- **최소 3개 이상의 기능별 클래스** 요구사항을 5개 기능 클래스로 충족.

### 1.2 클래스 목록 (7개)

| 클래스 | 책임 | 핵심 메서드 |
| --- | --- | --- |
| `MyKeyPair` | RSA 키 생성·저장·복구 (Key Management) | `generateAndSave`, `loadPublicKey`, `loadPrivateKey` |
| `CryptoUtil` | AES 키 생성·복구, AES 암복호화, SHA-256 | `generateAESKey`, `encryptAES`, `decryptAES`, `sha256`, `verifyHash` |
| `EnvelopeUtil` | 전자봉투 생성/열기 (AES키 ↔ RSA) | `createEnvelope`, `openEnvelope` |
| `SignatureUtil` | 전자서명 생성/검증 (SHA256withRSA) | `sign`, `verify` |
| `FileUtil` | 이진/문자열 파일 저장·읽기 | `saveBytes`, `readBytes`, `saveString`, `readString` |
| `EvaluationSubmit` | 학생 제출 프로그램 (main) | `main`, `run` |
| `EvaluationRead` | 교수자 열람 프로그램 (main) | `main`, `run` |

### 1.3 클래스 관계도

```
            ┌──────────────────┐         ┌──────────────────┐
            │ EvaluationSubmit │         │  EvaluationRead  │
            │   (학생 제출)     │         │   (교수자 열람)   │
            └────────┬─────────┘         └────────┬─────────┘
                     │  사용                        │  사용
        ┌────────────┼─────────────┬───────────────┼────────────┐
        ▼            ▼             ▼                ▼            ▼
   ┌─────────┐  ┌──────────┐  ┌──────────────┐  ┌──────────────┐  ┌─────────┐
   │MyKeyPair│  │CryptoUtil│  │ EnvelopeUtil │  │ SignatureUtil│  │FileUtil │
   │ RSA 키  │  │ AES/해시 │  │  전자봉투     │  │  전자서명     │  │ 파일I/O │
   └─────────┘  └────▲─────┘  └──────┬───────┘  └──────────────┘  └─────────┘
                     │ bytesToAESKey  │
                     └────────────────┘  (봉투를 열어 AES 키 복구)
```

### 1.4 전자봉투 핵심 흐름 (제출 → 열람)

```
[제출]  평가내용 ──AES암호화──> evaluation_001.enc
         AES키  ──RSA(교수자 공개키)──> envelope_professor_001.bin   ← 전자봉투
         암호문 ──SHA-256──> hash_001.txt
         검증정보 ──학생 개인키 서명──> signature_001.bin

[열람]  envelope ──RSA(교수자 개인키)──> AES키 복구   ← Key Recovery
         hash 재계산·상수시간 비교 → 무결성 확인
         서명 검증(학생 공개키) → 제출자 인증
         evaluation.enc ──AES복호화──> 평가내용 출력
```

## 2. 사용자 인터페이스 (콘솔 UI)

PDF의 "콘솔창을 활용한 텍스트 입력, 텍스트 아트" 항목에 맞춘 **텍스트 기반 UI** 이다.
구분선(`====`)과 단계 표시(`[1]`~`[7]`)로 진행 상황을 직관적으로 보여 준다.

### 2.1 학생 제출 화면 (`EvaluationSubmit`)

```
========================================
 전자봉투 기반 익명 동료평가 제출 시스템
 [학생 제출 모드]
========================================

학번 입력: 20211234
학교 계정 인증 중...
수강생 여부 확인 완료
제출 권한 확인 완료

익명 제출 ID 생성: SUBMITTER-DCEFF1

과목명 입력: 웹과코드보안
팀 번호 입력: 7
평가 대상자 입력: 홍길동
기여도 점수 입력(1~5): 4
서술형 평가 입력: 성실하게 참여했고 "핵심" 기능을 맡음
증빙자료 파일명 입력: evidence.png
제출 시간: 2026-06-08 11:54:05

[1] 평가 패키지 생성 완료
[2] AES 대칭키 생성 완료
[3] 평가 패키지 암호화 완료 -> evaluation_001.enc
[4] 전자봉투 생성 완료 -> envelope_professor_001.bin
[5] SHA-256 해시 생성 완료 -> hash_001.txt
[6] 전자서명 생성 완료 (SHA256withRSA) -> signature_001.bin
[7] 서버 저장 완료
평가 제출이 완료되었습니다. (교수자만 열람 가능)
========================================
```

- **입력 검증 UX**: 학번/팀번호에 숫자가 아닌 값, 점수에 1~5 밖의 값을 넣으면
  `-> 숫자만 입력하세요.` / `-> 1~5 사이의 숫자를 입력하세요.` 안내 후 재입력을 받는다.

### 2.2 교수자 열람 화면 (`EvaluationRead`)

```
========================================
 [교수자 열람 모드]
========================================

열람할 평가 번호 입력: 001
교수자 개인키 파일명 입력: professor_private.key

[1] 저장된 평가 데이터 불러오기 완료
[2] 전자봉투 복호화 완료 (교수자 개인키로 AES 키 복구)
[3] 평가 데이터 해시 검증
 - 해시 검증 결과: true
학생 공개키 파일명 입력: student_public_001.key
[4] 전자서명 검증 (SHA256withRSA)
 - 서명 검증 결과: true
[5] 평가 데이터 복호화 완료

========================================
 평가 내용
========================================
{ "anonymousSubmitterId":"SUBMITTER-DCEFF1", "course":"웹과코드보안",
  "team":"7", "target":"홍길동", "score":4, ... }
========================================
검증 결과:
 - 데이터 무결성: 정상
 - 제출자 전자서명: 정상
 - 열람 권한: 교수자 인증 완료
========================================
```

- **실패 시 UX**:
  - 데이터 변조 → `해시 검증 결과: false` 후 열람 중단.
  - 잘못된 교수자 개인키 → `열람 처리 중 오류가 발생했습니다: BadPaddingException` (스택 비노출, 안전한 일반 메시지).
  - 서명 불일치 → `전자서명이 유효하지 않습니다` 후 반영 중단.

### 2.3 실행 방법

```bash
cd src
javac -encoding UTF-8 *.java
java MyKeyPair          # 1) 학생·교수자 RSA 키쌍 생성
java EvaluationSubmit   # 2) 학생: 평가 입력·제출
java EvaluationRead     # 3) 교수자: 복호화·검증·열람
```
